1.基础知识：

1.1编码：

什么是编码？

编码有字符编码，数据编码，程序编码等，这里指的是字符编码：指将人类可读的字符转换为计算机可理解的二进制数据的过程。

  • ASCII：早期编码标准，用七位表示128个字符（英文字母、数字、符号）。
字符 ACSCII码（十进制） ACSCII码（十六进制） ACSCII码（二进制）
A 65 0x41 01000001
a 97 0x61 01100001
0 48 0x30 00110000
! 33 0x21 00100001
空格 32 0x20 00100000
换行符 10 0x20 00001010
  • UTF-8：目前最流行的Unicode编码方式，用1-4字节表示全球几乎所有文字。默认情况下Python3的源文件以UTF-8编码，也可以为源文件指定不同编码。
  • GBK/GB2312：中文常用的编码标准。（GB2312 收录约 6763 个常用汉字，GBK 收录约 21003 个汉字，兼容 GB2312）

乱码到底是啥？？？

原因：

乱码的本质是编解码用的字符集不一致。比如一段 UTF-8 编码的中文，被当成 GBK 去解码，字节映射到的就是错误字符，显示成一堆看不懂的符号（烫烫烫、锘挎垜等）。反之 GBK 当 UTF-8 读也会乱。

避免乱码的三条原则：

  • 写/读文件时显式指定 encoding（推荐 UTF-8）。
  • 终端、编辑器、源码三处编码保持一致。
  • 网络/文件传输时确认对方声明的编码。

1.2 标识符与保留字：

标识符是变量/函数/类的名字：字母或下划线开头，后接字母、数字、下划线，区分大小写。不能用保留字（keyword），也不能数字开头。

约定（PEP 8）：类名 CamelCase、函数与变量 snake_case、常量全大写下划线。单下划线 _foo 是”私有”约定；双下划线 __foo 会触发 name mangling。

2.数据类型：

变量无需声明类型，赋值即创建。内置标量：int / float / bool / complex；容器：str / list / tuple / set / dict。

2.1 数字 Number：

  • complex：3+4j。

15.标准库精选（二）：

15.1 struct —— 二进制数据解析：

这是安全方向的硬核技能。 网络包解析、二进制文件逆向、协议分析全靠它。它能在 Python 字节串和 C 语言结构体之间互相转换。

15.2 threading —— 多线程：

多线程用于并发执行 I/O 密集型任务（如同时扫描多个端口）。但要注意 Python 的 GIL 限制——CPU 密集型任务用 multiprocessing 更好。

15.3 logging —— 日志记录：

安全工具必须有日志。 入侵检测、审计追踪、调试排错都依赖规范的日志输出。比 print 强一万倍。

15.4 collections —— 高级容器：

比内置 list/dict/set 更强大的数据结构，安全分析中经常用到。

16.虚拟环境与包管理：

这一章对你来说是最实用的——你已经在服务器上用 conda/venv 隔离环境了，这里讲清楚原理和标准做法。

16.1 为什么需要虚拟环境？

不同项目依赖不同版本的包。如果全局安装，项目 A 需要 numpy==1.26，项目 B 需要 numpy==2.0，就会冲突。虚拟环境给每个项目一个独立的 Python + 包空间，互不干扰。

你服务器的约束就是典型例子：多人共享服务器，必须隔离环境，否则污染别人的工作空间。

16.3 pip 包管理：

安全提醒：

  • 永远不要 sudo pip install（Linux），这会污染系统 Python。
  • 安装前先确认包来源可信——供应链攻击是真实威胁（如 typosquatting 恶意包）。
  • 生产环境锁定版本（用 pip freeze > requirements.txt 定死版本号）。

17.浮点数精度问题：

浮点数在计算机中用二进制近似存储，不是精确值。这在 ML（梯度累积误差）和安全（密码学精度要求）中都可能踩坑。
